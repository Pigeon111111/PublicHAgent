# 微信小程序迁移可行性分析报告

## 一、项目现状分析

### 1.1 当前架构

**PubHAgent** 是一个基于 LangGraph 的公共卫生数据分析智能体系统，采用前后端分离架构：

```
┌─────────────────────────────────────┐
│  前端层 (Vue 3 + TypeScript)        │
│  - Element Plus UI                  │
│  - WebSocket + REST API             │
│  - ECharts 可视化                   │
└─────────────────────────────────────┘
              ↕ (HTTP/WSS)
┌─────────────────────────────────────┐
│  API 网关层 (FastAPI)                │
│  - WebSocket 网关                   │
│  - RESTful API                      │
└─────────────────────────────────────┘
              ↕
┌─────────────────────────────────────┐
│  核心工作流层 (LangGraph)           │
│  - Intent → Planner → Executor      │
│  - Reflection 循环                  │
└─────────────────────────────────────┘
              ↕
┌─────────────────────────────────────┐
│  Agent 层 + 工具层 + 沙箱           │
│  - 记忆管理 (mem0+ChromaDB)         │
│  - 工具系统 (内置+MCP+ 技能)         │
│  - Docker 安全执行环境              │
└─────────────────────────────────────┘
```

### 1.2 核心技术栈

**后端:**
- FastAPI + LangGraph + LangChain
- mem0ai (记忆系统) + ChromaDB (向量存储)
- pandas/numpy/scipy (数据分析)
- Docker (沙箱环境)

**前端:**
- Vue 3 + TypeScript + Vite
- Element Plus (UI 组件)
- Pinia (状态管理)
- ECharts (图表)
- WebSocket + Axios

#### 3.1.4 服务层适配

**WebSocket 服务适配:**
```typescript
// 原 frontend/src/services/websocket.ts
const ws = new WebSocket('ws://localhost:8000/ws/session-1')

// 改造为小程序版本
// src/services/websocket.ts
import Taro from '@tarojs/taro'

class WebSocketService {
  private socketTask: Taro.SocketTask

  connect(sessionId: string) {
    this.socketTask = Taro.connectSocket({
      url: `wss://your-domain.com/ws/${sessionId}`,
      protocols: ['chat']
    })
  }

  send(data: any) {
    this.socketTask.send({
      data: JSON.stringify(data)
    })
  }

  onMessage(callback: (msg: any) => void) {
    this.socketTask.onMessage((res) => {
      callback(JSON.parse(res.data))
    })
  }
}
```

**API 服务适配:**
```typescript
// 原 frontend/src/services/api.ts
import axios from 'axios'
const api = axios.create({ baseURL: '/api' })

// 改造为小程序版本
// src/services/api.ts
import Taro from '@tarojs/taro'

export function uploadFile(file: File) {
  return new Promise((resolve, reject) => {
    Taro.uploadFile({
      url: 'https://your-domain.com/api/upload',
      filePath: file.path,
      name: 'file',
      success: (res) => resolve(JSON.parse(res.data)),
      fail: (err) => reject(err)
    })
  })
}

export function request<T>(options: any): Promise<T> {
  return Taro.request({
    ...options,
    url: `https://your-domain.com${options.url}`
  }).then(res => res.data)
}
```

### 3.4 安全改造

#### 3.4.1 小程序登录认证

**实现微信登录流程:**
```python
# backend/api/routes/auth.py
import requests
from fastapi import APIRouter, HTTPException

router = APIRouter()

@router.post("/wechat/login")
async def wechat_login(code: str):
    """
    微信小程序登录接口
    1. 使用 code 换取 openid
    2. 创建或查找用户
    3. 生成自定义登录态（token）
    """
    appid = config.WECHAT_MINIPROGRAM_APPID
    secret = config.WECHAT_MINIPROGRAM_SECRET

    # 调用微信 API
    url = "https://api.weixin.qq.com/sns/jscode2session"
    params = {
        "appid": appid,
        "secret": secret,
        "js_code": code,
        "grant_type": "authorization_code"
    }

    response = requests.get(url, params=params)
    result = response.json()

    if "openid" not in result:
        raise HTTPException(400, "登录失败")

    # 查找或创建用户
    user = get_or_create_user(result["openid"])

    # 生成 token
    token = generate_token(user.id)

    return {
        "token": token,
        "user": {
            "id": user.id,
            "nickname": user.nickname,
            "avatar": user.avatar
        }
    }
```

## 五、技术难点与解决方案

### 5.1 WebSocket 连接限制

**问题:**
- 微信小程序要求 WSS 协议
- 最多同时 5 个 WebSocket 连接
- 需要域名备案

**解决方案:**
```python
# 使用 Nginx 反向代理配置 WSS
server {
    listen 443 ssl;
    server_name your-domain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location /ws {
        proxy_pass http://localhost:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }
}
```

### 5.2 文件上传适配

**问题:**
- 小程序文件 API 与 Web 不同
- 文件大小限制 100MB
- 需要用户授权

**解决方案:**
```typescript
// 封装文件上传
async function uploadFile(file: any) {
  // 1. 选择文件
  const res = await Taro.chooseMessageFile({
    count: 1,
    type: 'file',
    extension: ['csv', 'xlsx', 'json']
  })

  // 2. 上传文件
  const uploadRes = await Taro.uploadFile({
    url: 'https://your-domain.com/api/upload',
    filePath: res.tempFiles[0].path,
    name: 'file'
  })

  return JSON.parse(uploadRes.data)
}
```

### 5.3 图表渲染性能

**问题:**
- 小程序 Canvas 性能有限
- 复杂图表渲染卡顿
- 内存限制

**解决方案:**
1. **简单图表**: 使用 echarts-for-weixin
2. **复杂图表**: 服务端渲染为图片
3. **交互图表**: 限制数据量，使用简化版

### 5.4 Markdown 渲染

**问题:**
- 小程序不支持 dangerouslySetInnerHTML
- 代码高亮困难
- 数学公式渲染

**解决方案:**
```typescript

### 5.5 小程序包体积限制

**问题:**
- 主包限制 2MB
- 总包限制 20MB

**解决方案:**
1. **分包加载**: 非核心页面放入分包
2. **图片资源**: 使用 CDN
3. **按需引入**: UI 组件按需加载
4. **代码压缩**: 使用 terser 压缩

## 七、风险与建议

### 7.1 技术风险

1. **WebSocket 稳定性**
   - 风险：小程序 WebSocket 连接不稳定
   - 对策：实现自动重连机制，降级为轮询

2. **性能问题**
   - 风险：复杂图表渲染卡顿
   - 对策：服务端渲染图片，限制数据量

3. **兼容性问题**
   - 风险：不同微信版本兼容性
   - 对策：充分测试，使用基础 API

### 7.2 政策风险

1. **小程序审核**
   - 风险：数据分析类小程序审核严格
   - 对策：提前了解审核规范，准备资质

2. **数据合规**
   - 风险：公共卫生数据敏感性
   - 对策：数据脱敏，符合隐私政策

### 7.3 建议

1. **分阶段实施**
   - 先实现核心对话功能
   - 逐步添加文件管理、图表展示
   - 最后优化用户体验

2. **MVP 策略**
   - 第一版本只保留核心功能
   - 验证用户需求后再扩展

3. **技术预研**
   - 提前验证 echarts-for-weixin 性能
   - 测试 WebSocket 在弱网环境表现

## 八、总结

### 8.1 可行性结论

**✅ 完全可行**，PubHAgent 项目迁移到微信小程序具有高度可行性：

- 后端核心逻辑可 100% 复用
- 前端工作量约 43-61 人天
- 技术难度中等，无不可逾越障碍
- 成本可控，运维成本约 800-1700 元/月

### 8.2 核心优势

1. **架构优势**: 前后端分离，后端完全保留
2. **技术优势**: LangGraph 工作流无需改动
3. **成本优势**: 70% 代码可复用，开发成本低

### 8.3 关键改造点

1. **前端**: Vue 3 → 小程序（Taro 框架）
2. **通信**: WebSocket 适配（WSS + 鉴权）
3. **UI**: Element Plus → TDesign/NutUI
4. **图表**: ECharts → echarts-for-weixin
5. **登录**: 添加微信小程序登录

### 8.4 推荐实施路线

第 3-4 周：小程序框架 + 基础组件
第 5-8 周：核心页面开发
第 9-10 周：集成测试 + 性能优化
第 11-12 周：上线准备 + 提交审核
```
