# PubHAgent 用户手册

## 1. 系统介绍

PubHAgent 是一个针对公共卫生数据分析场景特化的智能 Agent 系统，基于 LangGraph 构建后端，配备美观、高可用性的前端界面。系统采用 Planner-Executor 范式，结合 Reflection 机制，实现复杂的数据分析任务自动化执行。

### 核心功能
- **智能意图识别**：准确识别用户数据分析需求
- **自动化分析流程**：通过 Planner-Executor 范式实现复杂分析任务的自动规划与执行
- **安全可控执行**：在沙箱环境中执行代码，确保系统安全
- **记忆系统**：支持用户偏好、分析方法、数据特征的持久化记忆
- **实时交互**：基于 WebSocket 的流式输出和 Human-in-the-loop 能力
- **可扩展工具系统**：支持 Tool、MCP、Skill 三层工具体系

## 2. 快速开始

### 环境要求
- Python 3.10+
- Node.js 18+
- Docker

### 安装步骤
1. **克隆代码库**
   ```bash
   git clone <repository-url>
   cd PubHAgent
   ```

2. **配置后端环境**
   ```bash
   # 创建虚拟环境
   python -m venv venv
   .\venv\Scripts\Activate.ps1  # Windows
   # 或 source venv/bin/activate  # Linux/Mac
   
   # 安装依赖
   pip install -r requirements.txt
   
   # 配置环境变量
   copy backend\config\.env.example backend\config\.env
   # 编辑 .env 文件，填入 API Key 等信息
   ```

3. **配置前端环境**
   ```bash
   cd frontend
   npm install
   ```

4. **启动服务**
   - 后端：`python -m uvicorn backend.api.main:app --reload`
   - 前端：`npm run dev`

5. **访问系统**
   - 前端：http://localhost:5173
   - 后端 API：http://localhost:8000/docs

## 3. 功能说明

### 3.1 文件上传
- **支持格式**：CSV、XLSX、JSON、Word
- **最大文件大小**：50MB
- **操作步骤**：
  1. 点击"上传文件"按钮或拖拽文件到上传区域
  2. 等待上传完成
  3. 查看文件列表确认上传成功

### 3.2 数据分析
- **支持分析类型**：
  - 描述性统计
  - 回归分析
  - 生存分析
  - 数据可视化
  - 数据清洗
- **操作步骤**：
  1. 上传数据文件
  2. 在输入框中描述分析需求
  3. 点击"发送"按钮
  4. 等待 Agent 执行分析
  5. 查看分析结果

### 3.3 实时交互
- **流式输出**：实时显示 Agent 的思考过程和执行进度
- **打断功能**：点击"打断"按钮可中途停止分析
- **修正功能**：可以修改分析计划并重新执行

### 3.4 历史记录
- **对话历史**：查看过去的对话记录
- **分析历史**：查看历史分析结果
- **文件历史**：查看上传的文件

## 4. 常见问题

### 4.1 上传文件失败
- **可能原因**：
  - 文件格式不支持
  - 文件大小超过限制
  - 网络问题
- **解决方案**：
  - 确认文件格式正确
  - 压缩文件或分割大文件
  - 检查网络连接

### 4.2 分析结果不符合预期
- **可能原因**：
  - 描述不够清晰
  - 数据质量问题
  - 分析方法选择不当
- **解决方案**：
  - 更详细地描述分析需求
  - 检查数据质量
  - 尝试不同的分析方法

### 4.3 系统响应缓慢
- **可能原因**：
  - LLM 调用延迟
  - 沙箱启动时间
  - 数据分析复杂度高
- **解决方案**：
  - 简化分析需求
  - 优化数据格式
  - 耐心等待

### 4.4 错误提示
- **常见错误**：
  - API Key 错误：检查环境变量配置
  - 沙箱启动失败：检查 Docker 服务
  - 文件解析错误：检查文件格式
- **解决方案**：
  - 按照错误提示进行修复
  - 参考文档中的配置说明
  - 联系系统管理员
